ğŸ› ï¸ Common Issues & Easy Fixes

Even superheroes need help sometimes! Here are the most common VS Code Git issues students face and how to fix them quickly.

ğŸ˜µ "I Don't See the Source Control Panel!"

Problem: Student can't find Git features in VS Code.

Quick Fix:

  1. Make sure you opened a folder, not just a single file
  2. Check if the folder contains a .git folder (it might be hidden)
  3. If no Git repo exists, click "Initialize Repository" in the Source Control panel
Prevention:
Always use "Open Folder" instead of "Open File"

ğŸ”’ "VS Code Wants My GitHub Password!"

Problem: Student can't push to GitHub due to authentication.

Quick Fix:

  1. Install the "GitHub" extension in VS Code
  2. Click "Sign in to GitHub" when prompted
  3. Complete the browser authentication
  4. VS Code will remember your login!
Teacher Note:
This is much easier than setting up SSH keys for beginners!

ğŸ˜± "I Accidentally Committed the Wrong Thing!"

Problem: Student made a commit they want to undo.

Quick Fix:

  1. In Source Control panel, click the "..." menu
  2. Select "Undo Last Commit"
  3. Your changes go back to being "uncommitted"
  4. Fix what you need, then commit again!
Relief:
Nothing is ever truly lost in Git - you can always undo!

ğŸ¤” "My Teammate's Changes Aren't Showing Up!"

Problem: Student can't see recent changes from GitHub.

Quick Fix:

  1. Click the â¬‡ï¸ pull button in the Source Control panel
  2. Or use the "..." menu and select "Pull"
  3. VS Code downloads all new changes from GitHub
  4. Refresh your file explorer to see updates!
Habit:
Always pull before starting work each day!
